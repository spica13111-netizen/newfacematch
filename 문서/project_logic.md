# 구글 발주서 자동화 프로그램 로직 정리

이 문서는 프로그램의 작동 로직과 요구사항을 정리한 것입니다.

## 1. 인증 (Authentication)
- **방식**: 실행 파일과 **동일한 폴더**에 있는 서비스 계정 JSON 파일을 자동으로 찾아서 사용합니다.
- 사용자에게 파일을 선택하게 하는 팝업을 띄우지 않습니다.
- **필요한 권한**: Google Sheets API만 필요 (Drive API 불필요)

## 2. 스프레드시트 URL 관리
- Drive API 
## 3. 탭(Sheet) 별 로직

### A. 'daily' 탭 (일일 발주)
- **헤더 파싱 로직**:
    1. **4행(Row 4)**을 읽습니다.
    2. **A열(Column A)**에는 반드시 `"입력불가"`라는 텍스트가 존재합니다.
    3. 실제 데이터 헤더는 **B열**부터 시작합니다.
    4. 헤더의 끝은 그 다음으로 `"입력불가"`가 나오는 열의 **바로 앞 열**까지입니다.
    5. 즉, 유효한 헤더 범위는 `B열 ~ (다음 '입력불가' 열 - 1)` 입니다.

### B. 'setting' 탭 (설정)bat
    1. 'daily' 탭에 입력된 일일 발주 정보를 읽습니다.
    2. 'raw' 탭으로 데이터를 이동(날짜별로 쌓음).
    3. 'daily' 탭의 입력 데이터를 초기화합니다.

## 4. 파일 위치
- 이 로직은 `h:/내 드라이브/뉴페이스 프로그램/(x)구글 발주서 데이터/` 경로에서 실행되는 것을 기준으로 합니다.
